# Generic PHP

All the rules listed here applies in general to all PHP code.

# Functions

These are the rules which are related to a function or method.

# Camel case naming

When creating a method the name should always be following camel case naming.

Bad

function my_private_method() {
    return 'private';
}

Good

function myPrivateMethod() {
    return 'private';
}

# Must declare return type

Always add return types.

Bad

function myMethod() {
    return 'works';
}

Good

function myMethod(): string {
    return 'works';
}

# Void return types

If a method returns nothing, it should be indicated with void.
This makes it more clear to the users of your code what your intention was when writing it.

Bad

function handle()
{
    // Do something
}

Good

function handle(): void
{
    // Do something
}

# Required visibility

When defining a method, the visibility must be declared also.

  • abstract and final MUST be declared before the visibility
  • static MUST be declared after the visibility.

Bad

class Example
{
    static final function works(): void
    {
        // Work work...
    }
}

Good

class Example
{
    final public static function works(): void
    {
        // Work work...
    }
}

# Max line length of functions

A function should not be longer than 20 lines long.

# Classes

All the rules specific for a class will be listed here.

# One class per file

There should only be one class per file. This helps simplify the files and makes finding a class faster.

# One interface per file

# One trait per file

# Max properties in a class

A class should not contain more than 10 properties.

# Max methods in a class

A class should not contain more than 10 methods.

# Max line length of classes, traits and interfaces

A class, interface of trait should not be longer than 200 lines long.

# Code

# Declare strict

It is recommended to add declare(strict_types=1) as it can fix some unexpected results when type hinting.
Without strict it allows type coercion, meaning bool will be casted to int for example.

# Trailing commas on arrays

When defining an array, we should always have a trailing comma as long as the array is multiple lines.
When dealing with single line arrays, it is not required, however is allowed.

Bad

$array = [
    'test' => 'bad'
];

Good

$array = [
    'test' => 'bad',
];

// Or
$array = ['test' => 'bad'];

# Max length of lines

A line should not be longer than 80 characters long.

# Documentation

# No redundant @param phpdoc annotations

If the native method declaration contains everything and the phpDoc does not add anything useful then the annotation can be removed as it is redundant.

Bad

/**
 * @param User $user
 */
public function handle(User $user): void
{
    // Do something on the user
}

Good

public function handle(User $user): void
{
    // Do something on the user
}
